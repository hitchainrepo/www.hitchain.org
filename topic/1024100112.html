HitChain Project Biweekly Report:11 February—24 February, 2019

February 24, 2019

1.Project Progress

1) Create Hit-Java core modules and migrate existing functional code:

A. Adding the module of org. hitchain. core based on the development branch, the core function code of Hitchain will be developed under this module to reduce the degree of JGit source code intrusion and facilitate the later JGit source code upgrade;

B. Migrate the completed Hitchain-related code in Hitbucket into this module to reuse the functions that have been developed;

C. Refactoring the migrated code, standardizing the naming of tool classes, correcting code errors in migration, and correcting project dependency problems in migration;

D. refactoring Git tool class, adding onPush, onFetch, onClone and other functional entries, the main functions of hitchain will be injected into these entries.

2) Continue to study the multiple submission mechanism of HIT:

A. Continue to study and verify the design of user account information storage and account switching for HIT multiple submission mechanism to ensure that users switch accounts on multiple GIT services for multiple submissions;

B. Continue to study and validate the submission content interpretation of HIT multiple submission mechanism, and separate Commit content for multiple submission.
