HitChain Project Biweekly Report:23 September—6 October, 2019

6 October, 2019

I. Technological Research and Development

1) Add a warehouse hit1.0-specs:

A. Add a hit1.0-specs repository to store all specification documents;

B. Organize the contents of the Project Information File specification, detailing the structure and usage of the document;

C. Organize the Git File Index specification to specify the structure and usage of the document;

D. Warehouse location: https://github.com/hitchainrepo/hit1.0-specs.git.

2) Warehouse Browser Development:

A. New project information details page, browse the basic information such as the creator and member of the project;

B. The Ethereum block browsing service in the warehouse browser is switched from infura web3j to etherscan api, which solves the problem that the infura service may not be accessible to the network in China;

II. Development Plan

1) September plan

A. Implement code encryption for PullRequest;

B. Implement a code repository browser that retrieves code repository project information on the blockchain;

2) Follow-up plan

A. Implement the Pull Request browser to support visualization Pull Request browsing;

B. Implement a hit wallet for managing the owned warehouse, HIT token transactions, ETH token transactions.
