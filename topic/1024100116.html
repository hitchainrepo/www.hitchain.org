HitChain Project Biweekly Report:8 April—21 April, 2019

21 April, 2019

1.Project Progress

1) Improve the documentation of hit usage:

A. Perfect documentation includes: construction and installation, instructions for use, interface instructions, problems and support, developers, etc.

2) Implementing the function of private warehouse:

A. Implementing the management function of team members.

B. Add team member management command hit member [add | remove], which can add or delete team members

C. Implementing the management of private warehouse encryption key pairs.

D. Add the private warehouse key pair management command hit keypair [add | remove | renew], which can add, delete or update key pairs. At the same time, after changing the key pair, the warehouse will be resubmitted. If the warehouse has configured the key pair, the warehouse will be re-encrypted and submitted. If the warehouse key pair has been removed, the encryption will be removed and resubmitted.

E. Implement the functions of generating, encrypting and decrypting warehouse key pairs and encrypting and decrypting warehouse files.

3) Fix bugs in hit-java public warehouse and its original functions to support private warehouses:

A. The problem of revising ECC encryption algorithm is compatible with private warehouse encryption algorithm.

B. Delete Base64 encoding content in RSA encryption algorithm toolkit to ensure that Hex is used globally for unified encoding to reduce the confusion of encoding usage.

C. Modify the stream reading mode of file encryption and decryption to avoid incomplete read data caused by multiple reads of the same input stream.

4) Hit public warehouse command function test:

A. Testing the function of hit command.
