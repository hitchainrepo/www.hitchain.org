HitChain Project Biweekly Report:25 February—10 March, 2019

March 24, 2018

1.Project Progress

1) Implementing the functions of Hit-Java Public Warehouse:

A. Extending JGit's transport protocol. The HIT protocol is extended on the basis of JGit transport protocol interface. The URLs of the transport protocol begin with hit://. The HIT protocol will be used only when the HIT protocol URL is used. The original functions of other JGit protocols are still used when the URLs of the HIT protocol are not used.

B. Extend the object database based on IPFS. Based on IPFS, Git object database is extended to WalkRemoteObject Database in JGit to realize related interface functions. When Git files and Objects are uploaded and downloaded, content is accessed through IPFS.

C. Extend the storage mechanism of object database based on IPFS. GitFileIndex protocol content in HIT protocol is implemented based on IPFS object database, which ensures that files can be indexed when uploading content and avoids duplicate uploading of duplicate content, so as to improve uploading speed. Integrating the function of interacting with Taifang Intelligent Contract, getting the latest project address and pulling project Metadata through the project address in the Intelligent Contract;

D. Extending HIT URI. Extend the URIish in JGit to support HIT URI. In order to get the latest address of HIT project Clone, the smart contract address is used to get the latest address of the project, and the project Metadata is pulled to get the name of the project, then the Clone operation is performed.

2) Continue to study the multiple submission mechanism of HIT:

A. Continue to study and verify the design of user account information storage and account switching for HIT multiple submission mechanism to ensure that users switch accounts on multiple GIT services for multiple submissions;

B. Continue to study and validate the submission content interpretation of HIT multiple submission mechanism, and separate Commit content for multiple submission.
