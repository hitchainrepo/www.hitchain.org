HitChain Project Biweekly Report:17 June—30 June, 2019

30 June, 2019

1.Project Progress

I. Technological Research and Development

1) Implementing the functions of different code warehouses and Pull Request to migrate to HIT by one key:

A. Implement hit migrate command, which supports the migration of Git warehouse to HIT by one key. At present, Gitee and Github public warehouses of code cloud are supported, and Pull Request of code cloud Gitee and Github are migrated together in the process of migrating warehouse code.

B. To implement the function, at present, hit does not support git am command function, but still uses git am command to operate in the process of Pull Request merging, which will be followed by hit am command; empty Patch will occur in Merge PR in Pull Request, which will be tested and dealt with later;

2) Optimize the operation of intelligent contract, reduce the input of parameters in the command process, and simplify the mission order.

A. Adding hit CFG gas command function, which can set up Gas Limit and Gas Wei needed for contract deployment and writing, and reduce the need to input these two parameters in the process of intelligent contract deployment and writing;

B. Repository Name Warehouse Intelligent Contract and PullRequest Intelligent Contract Command were optimized, and Gas Limit and Gas Wei parameters were removed.

3) Optimize the function of Patch in Pull Request:

A. Reconstructing Patch function and putting related functions into Patch Helper, which can automatically process and generate Summary information of Patch, compatible with Gitee and Github of code cloud, and support Git's native standard;

B. Support the merging of multiple Pull Requests into a single list to reduce the interaction of smart contracts.

II. Development Plan

1) July Plan

A. Implementing hit am functions and code Pull Request merging functions and processes to provide better operational experience;

B. Optimize intelligent contracts and minimize contract interactions.

2) Follow-up plan

A. Implementing the code warehouse browser, which can retrieve the project information of the code warehouse on the block chain;

B. Implement Pull Request browser to support visual Pull Request browsing.

C. Realize the hit wallet to manage the warehouse, the HIT token transaction and the ETH token transaction.
