HitChain Project Biweekly Report 15 September, 2018—30 September, 2018

September 30, 2018

1. News Review

1）Tyler Chen, senior architect of HitChain, was invited to participate in The Computing Conference 2018 GBLS Session on September 20th. At the conference, Mr. Chen introduced the development plan and project progress of HitChain in the current environment, and conducted an in-depth analysis of the underlying code of HitChain.

2）On September 26, HitChain's senior chief architect Tyler Chen attended the BlockDAO product launch conference. At the conference, Cloud Town Management Committee, BlockDAO, Blockchain Research Center in School of Electronics Engineering and Computer Science Peking University, LD Research, BlockLabs and many other enterprises and institutions jointly launched the Yunxi Blockchain Industry Development Association. In addition, BlockDAO has also established an incubator——DAOSpace to support the blockchain quality project. HitChain, as the first representative of the incubator project, will accelerate the application of technology and improve the ecological construction with the assistance of BlockDAO.

2. Project Progress

1) Refactoring the services that go-ipfs depend on and separating them from the managed dependencies in the native network. Provides an efficient private network by loading them from the network quickly and independently.

2) Add a mechanism to generate customized swarm.key for ipfs, so that to make it connect to HIT private network in initialization phase.

3) Extending "ipfs init" instruction to add two customized parameters to binding ipfs and hit foundation's disaster preparedness servers dynamically.

4) Implement the monitoring module of HIT storage network services.

A. In the aspect of private storage network services, this module can monitoring the initialization (init) action of user nodes and identify the normal initialization requests and add them to the server's services list.

B. In the aspect of authorization control of collaborative development, this module cat monitoring the user's add action and determines whether the user node has the right to modify or access the contents.

C. Realize the threads of the server's monitoring dynamically scheduling and release to prevent from DOS attacks.

5) Due to the delay in IPFS research and development, the HIT test is delayed accordingly.

6) Checkout and clean Gitlab branch to keep the branch 11.3.0 source codes and committed to repository as initialization version, the repository address: https://github.com/hitchainrepo/hitlab.

7) Build the local development environment to test the source code of branch 11.3.0 to ensure work right.

8) Build the public development environment to test the source code of branch 11.3.0 to ensure work right for collaborative development.

9) Draw up the 0.1 version of HIT protocol in the form of the Swagger API, the API file address: https://github.com/hitchainrepo/hitlab/blob/master/doc/hitlab/api/hitapi-swagger.json.

10) There are 52 commits by ipfs refactoring and 6 commits by hitlab.

3. Market News

1) From September 14th to 15th, HitChain senior architect Jerry Li met with the cryptocurrency service provider X Infinity project team at the Jeju World Blockchain Forum (WBF) and reached initial strategic cooperation with them. X Infinity which has cooperated with Master and UnionPay will launch its e-wallet and HitChain is expected to list on the e-wallet. In the future, it can directly purchase HIT coins by card. HitChain will continue to collaborate with outstanding blockchain projects to enrich its eco-partners.
