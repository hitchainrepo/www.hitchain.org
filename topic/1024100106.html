HitChain Project Biweekly Report:17 November—2 December, 2018

December 02, 2018

1. News Review

1) From November 19th to 20th, HitChain was invited to participate in 2018 Blockchain New Economy Hangzhou Summit. The summit was based on the theme of “Technology Driving Economy”，hosted by the Hangzhou Government. There were 24 keynote speeches and 6 roundtable forums in the summit. Tyler Chen, the senior architect of HitChain, took part in the conference with the technical team and joined the 2018 Blockchain Pioneer List to win the “Growth Pioneer” award.

2. Project Progress

1) Design the plan for concurrent testing of Hithub: using kubernetes to control dockers to connect Hithub, and executing Hit operation.

2) Design the plan for the use of ipfs api and hit api: taking the Windows, Linux, Mac operating systems into consideration.

3) Design the improvement plan for zero-knowledge verification:

a. Randomly generating a temporary public-private key pair when starting the daemon process of ipfs;

b. By using private key signature and public key verification, we can prevent the content from being tampered;

c. The server needs to update the temporary public key of client nodes at any time.

4) Design the plan for initializing clients dynamically according to the network delay in multi-server node situation.

a. Pre-fetch the network delay of each service node;

b. Automatically select a server according to the network delay by default;

c. Clients can designate a server manually.

5) According to the designed HIT storage protocol, a version based on centralized managed code is implemented.

6) Design HIT API interface and present some functions.

7) Convert part of Scala code to Java code.
