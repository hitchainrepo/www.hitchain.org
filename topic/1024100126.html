HitChain Project Biweekly Report:1 July—14 July, 2019

14 July, 2019

1.Project Progress

I. Technological Research and Development

1) Implement Pull Request Patch merging function:

A. Implementing hit am command to support direct merging of Pull Request through this command, while generating pull request branch can be used to manually merge branch to the main branch;

B. To realize the function of reserving the original commit information in application patch, and to support the automatic creation of commit according to the commit information in application patch.

2) Optimize the Intelligent Contract Merging Repository Name Intelligent Contract and Pull Request Intelligent Contract Function:

A. Implement Hit Repository Intelligent Contracts, merge Repository Name Intelligent Contracts and Pull Request Intelligent Contracts to reduce the interaction and deployment of Intelligent Contracts to save time and GAS consumption;

B. Implementing API and function realization of HitRepository intelligent contract;

C. Deploying and testing the functions of HitRepository intelligent contracts;

3) Make HIT video:

A. Make HIT one-click migration demonstration video, migrate existing warehouse through hit migrate one-click, video address: https://weibo.com/6625287274/HCwALAqTc;

B. Make a demonstration video of HIT using video to demonstrate the function of HIT using cycle. Video address: https://weibo.com/6625287274/HCWo71Vzq.

II. Development Plan

1) July Plan

A. Implementing hit am functions and code Pull Request merging functions and processes to provide better operational experience;

B. Optimize intelligent contracts and minimize contract interactions.

2) Follow-up plan

A. Implementing the code warehouse browser, which can retrieve the project information of the code warehouse on the block chain;

B. Implement Pull Request browser to support visual Pull Request browsing.

C. Realize the hit wallet to manage the warehouse, the HIT token transaction and the ETH token transaction.
