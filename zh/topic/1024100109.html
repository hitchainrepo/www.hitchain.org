HitChain 项目双周报（2018年第6期）：11月5日～11月16日

2018-09-15

HitChain运营团队受邀参加金色财经主办的第二期沙龙活动“金色沙龙论生态；HitChain专注于服务开源项目，提供去中心化代码托管，可与钱包的代码开源展开深度合作

1. 新闻回顾

1）11月9日下午，HitChain运营团队受邀参加金色财经主办的第二期沙龙活动“金色沙龙论生态 数字货币钱包的开源之路”。团队认为钱包开源是必然趋势，但需要逐步进行，HitChain专注于服务开源项目，提供去中心化代码托管，可与钱包的代码开源展开深度合作。

2. 项目进展

1）完成存储即挖矿的奖励机制设计：

a、客户端定时报备节点存储情况（例如，连续在线30分钟才算时长）：通过零知识验证方法，防止客户端节点的信息篡改；

b、 服务器定时检查客户端服务情况：定时检查客户端在线情况；

c、 服务器异常数据检测服务：定时检测客户端节点报备数据与服务端定期检查数据中存在的异常情况（根据报备时间间隔以及定期检查结果检测节点的异常行为）。

2）完成了客户端定时报备节点存储情况功能。

3）完成了服务器定时检查客户端服务情况功能。

4）修改了权限认证方式，使得hit支持分支操作。

5）完成了windows的兼容性测试，包含命令：

a、hit push；

b、hit transfer；

c、hit pull；

d、hit clone；

e、hit commit。

6）将部份 Sacal 代码转为 Java 代码，以实现 Java 扩展。

7）完善 HIT 存储协议，并编码实现索引部分功能。

8）研究与设计 DAPP 与 HIT 存储交互。
