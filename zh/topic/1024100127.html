HitChain 项目双周报（2019年第15期）：7月15日～7月28日

2019-7-28

1.项目进展

一、技术研发

1）优化智能合约合并 RepositoryName 智能合约及 Pull Request 智能合约功能：

a）测试及优化 HitRepository 智能合约功能，确保合并后的智能合约能兼容原有的 RepositoryName 及 PullRequest 智能合约的功能；

b）优化 HitRepository 智能合约添加权限及功能约束，智能合约拥有者可以更好地控制合约的功能开启及关闭；

c）优化 HitRepository 智能合约添加用户的个人信息功能，包括 name 及 email 等；

2）优化原有的代码，采用与JGit原代码兼容的设计方式重新设计 Hit：

a）采用基于 Command 的设计方式优化原有的 Hit 代码，可以直接通过 Hit 类直接创建相应的 Command 来对仓库及配置等进行操作；

b）优化 Main 入口类，采用 Hit Command 的方式来调用及使用相应的功能；

c）优化原的采用两个合约操作的逻辑，转为通过一个智能合约来完成以前所有的功能。

二、研发计划

1）8月份计划

a）优化智能合约，提升合约交互的效率；

b）实现 PullRequest 的代码加密。

2）后续计划

a）实现代码仓库浏览器，可检索区块链上的代码仓库项目信息；

b）实现 Pull Request 浏览器，支持可视化 Pull Request 浏览；

c）实现 hit 钱包，用于管理所属的仓库、 HIT 代币交易、ETH 代币交易。
