2. 项目进展

1）完成 HIT 与 IPFS 整合研究，实现 GIT 基本功能：

已完成基于 GIT 的项目存储到 IPFS 中，并且在命令兼容原生 GIT 的用法，基本命令包括：

a、hit transfer，把一个公有 GIT 仓库完整 clone 下来；

b、hit commit，提交更改；

c、hit push，提交本地更改到远程 IPFS 中；

d、hit clone，把远程仓库检出到本地文件系统；

e、hit add，添加文件到索引中；

f、hit pull，同步远程更改。

2）完成通过 IPNS 实现仓库地址的影射：

由于 IPFS 每次更改后，其 HASH 地址都会变动，不利于项目的持续更新和寻址。而通过 IPNS 机制，在创建项目时把项目地址影射到一个固定地址上，且后续更新都不用改变地址，有利于项目代码分享。

3）完成 HIT 简单的权限控制：

P2P 网络存储在权限控制上比较困难，但为了实现只允许仓库所有者及授权对象才能修改本仓库代码，以及多帐号对仓库的共同管理，而引入了非对称加密方法进行授权，对于多帐号的仓库管理采用非对称二次加密方法进行授权。对于多帐号的添加，提供了命令：hit add-user。

4）部分完成 HIT 可视化应用需求设计：

完成小部分HIT 可视化应用需求设计，主要是让用户从码云或 GITHUB 迁移到 HIT 时可以沿用部分必要的功能。

5）部分完成 HIT 可视化应用界面：

可视化应用界面主要用于应用设计的交流及收集意见，是基于 HIT 功能上扩展的一些常用功能。

3.市场动态

1.HitChain基金会战略合作伙伴开源中国与百度公司共同获得国家科学技术部“云计算和大数据”重点专项资金支持，将重点研究基于机器编程的智能开发云平台和基于代码分析查重的代码推荐技术。开源中国将与HitChain分享本次研究成果，并通过该项目成果应用来促进HitChain的底层代码查重和鉴权技术，加速HitChain项目在线开发和代码托管场景的落地。

2.澳洲创业交流社区NextGenius的区块链训练营会议上，HitChain基金会主席黎江介绍并推广了HitChain的愿景规划和代码技术，并与创业人士讨论了服务工具类Token和证券型Token的区别以及未来政策情况，在讨论中，大家普遍预计澳大利亚将在明年将证券型代币纳入监管体系并允许上市交易，该举措很可能会冲击澳大利亚股票市场。
