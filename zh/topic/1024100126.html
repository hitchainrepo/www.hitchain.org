HitChain 项目双周报（2019年第14期）：7月1日～7月14日

2019-7-14

1.项目进展

一、技术研发

1）实现 Pull Request Patch 合并功能：

a）实现 hit am 命令，支持通过该命令直接合并 Pull Request，同时生成 pull request branch 可用于手动合并 branch 到主分支上；

b）实现 apply patch 中保留原 commit 信息的功能，支持在 apply patch 过程中根据 patch 中的 commit 信息自动创建 commit；

2）优化智能合约合并 RepositoryName 智能合约及 Pull Request 智能合约功能：

a）实现 HitRepository 智能合约，合并 RepositoryName 智能合约及 Pull Request 智能合约功能，以减少智能合约的交互及部署以节省时间及 GAS 消耗；

b）实现 HitRepository 智能合约的 API 及功能实现；

c） 部署及测试 HitRepository 智能合约的功能；

3）制作 HIT 使用视频：

a）制作 HIT 一键迁移的演示视频，通过 hit migrate 一键迁移已有仓库，视频地址：https://weibo.com/6625287274/HCwALAqTc；

b）制作 HIT 使用视频的演示视频，演示 HIT 使用周期的功能，视频地址：https://weibo.com/6625287274/HCWo71Vzq。

二、研发计划

1）7月份计划

a）实现 hit am 功能及代码 Pull Request 合并功能和过程提供更好的操作体验；

b）优化智能合约，尽量减少合约的交互。

2）后续计划

a）实现代码仓库浏览器，可检索区块链上的代码仓库项目信息；

b）实现 Pull Request 浏览器，支持可视化 Pull Request 浏览；

c）实现 hit 钱包，用于管理所属的仓库、 HIT 代币交易、ETH 代币交易。
