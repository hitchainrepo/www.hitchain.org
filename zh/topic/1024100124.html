HitChain 项目双周报（2019年第12期）：6月3日～6月16日

2019-6-16

1.项目进展

一、技术研发

1）实现 Pull Request 功能：

a、添加 Pull Request 合约操作命令，激活 Pull Request 命令：enable；创建 PullRequest 命令：create；只读命令：owner,delegator,communityPullRequest,communityPullRequestCount,authedPullRequest,authedPullRequestCount,authedAccount,authedAccountList,authedAccountCount,listCommunityPR,listAuthedPR，其中listCommunityPR，listAuthedPR功能未实现；写入智能合约命令：changeOwner,delegateTo,addPullRequest,addAuthedAccount,removeAuthedAccount；

b）实现 Pull Request 的 Patch 功能，支持把多个 Commit 的内容写入到一个 Patch 中，支持保留已有的 Commit 的信息，支持通过 Git 的 am 命令导入 Pull Request 的 Patch；

2）测试 Pull Request 智能合约及修改问题：

a）修复 Pull Request 命令参数传递问题；

b）测试 Pull Request 生成的 Patch 的可用性和兼容性；

3）设计 Pull Request 信息内容：

a）设计 Pull Request 的详细信息内容数据结构，数据提交以及数据获取的关联关系。

二、研发计划

1）6月份计划

a、实现 Pull Request 基本功能，支持 Pull Request 的提交；

b、实现外部 Git 仓库一键导入功能，支持 Pull Request 导入。

2）后续计划

a、实现代码仓库浏览器，可检索区块链上的代码仓库项目信息；

b、实现 Pull Request 浏览器，支持可视化 Pull Request 浏览；

c、实现 hit 钱包，用于管理所属的仓库、 HIT 代币交易、ETH 代币交易。
