HitChain 项目双周报（2019年第17期）：8月12日～8月25日

2019-8-25

1.项目进展

一、技术研发

1）完善 hit migrate 迁移仓库过程异常：

a）完善 hit migrate 迁移仓库过程中 pull request 的 patch 可能为空的问题，一些仓库在特殊情况下 pull request 的 patch 为空，会导致在 patch 合并时无法合并该 patch 内容的问题；

b）完善 hit migrate 迁移仓库过程中 pull request 数量大的问题，一些仓库在迁移过程中，由于 open 状态的 pull request 数量较大，并且访问可能受限情况下会迁移失败的情况，通过添加参数允许用户在迁移仓库时可指定迁移的 pull request 的数量；

c）优化代码把迁移的主要逻辑合并到 MigrateCommand 中；

2）实现 pull request 代码加密：

a）对于一些加密的仓库，其 pull request 代码也同样添加加密的逻辑;

3）代码仓库浏览器设计：

a）设计代码仓库的浏览功能;

4）hit-java 项目仓库主分支由 master 切换到 dev-1.0。

二、研发计划

1）8月份计划

a）优化智能合约，提升合约交互的效率；

b）实现 PullRequest 的代码加密；

c）测试与修正某些 PullRequest 合并的问题。

2）后续计划

a）实现代码仓库浏览器，可检索区块链上的代码仓库项目信息；

b）实现 Pull Request 浏览器，支持可视化 Pull Request 浏览；

c）实现 hit 钱包，用于管理所属的仓库、 HIT 代币交易、ETH 代币交易。
