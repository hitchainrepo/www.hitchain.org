HitChain 项目双周报（2019年第13期）：6月17日～6月30日

2019-6-30

1.项目进展

一、技术研发

1）实现不同的代码仓库及 Pull Request 一键迁移到 HIT 的功能：

a) 实现 hit migrate 命令，支持通过该命令一键把 Git 仓库迁移到 HIT 上，目前支持码云的 Gitee 及 Github 的公开仓库，支持在迁移仓库代码过程中把码云 Gitee 及 Github 上的 Pull Request 一并迁移过来；

b）待实现功能，目前 hit 不支持 git am 命令功能，在进行 Pull Request 合并过程仍然使用 git am 命令来操作，后续会实现 hit am 命令；Pull Request 中有 Merge PR 时会出现空 Patch 的情况，后续会测试及处理该问题；

2）优化智能合约操作，减少命令过程中的参数输入，使命令更加精简：

a）添加 hit cfg gas 命令功能，通过该命令可以设置合约部署及写入所需要花费的 Gas Limit 及 Gas Wei，减少了在智能合约部署及写入过程中仍需要输入这两个参数；

b）优化了 RepositoryName 仓库智能合约及 PullRequest 智能合约命令，去除了 Gas Limit 及 Gas Wei 两个参数；

3）优化 Pull Request 中 Patch 的功能：

a）重构 Patch 功能，把相关的功能放到 PatchHelper 中处理，可以自动处理及生成 Patch 的 Summary 摘要信息，兼容码云的 Gitee 及 Github ，支持 Git 的原生标准；

b）支持将多个 Pull Request 合并成一个列表，以减少智能合约的交互。

二、研发计划

1）7月份计划

a、实现 hit am 功能及代码 Pull Request 合并功能和过程提供更好的操作体验；

b、优化智能合约，尽量减少合约的交互。

2）后续计划

a、实现代码仓库浏览器，可检索区块链上的代码仓库项目信息；

b、实现 Pull Request 浏览器，支持可视化 Pull Request 浏览；

c、实现 hit 钱包，用于管理所属的仓库、 HIT 代币交易、ETH 代币交易。
